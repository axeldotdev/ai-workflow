# Learn

Help the user learn and understand a new subject, technology, package, or concept. All output in English.

## Structure

Follow this progression for any topic:

### 1. What Is It?
- Clear, jargon-free definition
- One-sentence elevator pitch
- Category: what kind of thing is it? (library, framework, protocol, pattern, etc.)

### 2. Why It Matters
- What problem does it solve?
- What existed before it? What are the alternatives?
- When should you use it (and when should you NOT)?

### 3. Core Concepts
- Key terms and mental models
- Fundamental building blocks
- Use bullet points and ASCII diagrams to illustrate relationships

```
Concept A ──▶ Concept B ──▶ Concept C
                 │
                 ▼
            Concept D
```

### 4. Practical Examples
- Code snippets showing basic usage
- Start with the simplest possible example
- Build up to a realistic use case
- Annotate code with comments explaining each step

### 5. Resources
- Use WebSearch to find and provide real, up-to-date links:
  - Official documentation
  - Best tutorials or guides
  - GitHub repository (with stars count if relevant)
  - Key blog posts or videos
- Only include links you've verified via search

## Guidelines

- **Progressive depth**: Start with a clear overview. Go deeper only when the user asks.
- **Code snippets**: Include them whenever the topic involves code. Use the language/framework most relevant to the user's context.
- **ASCII diagrams**: Use them to explain architectures, flows, hierarchies, or relationships.
- **Analogies**: Use real-world analogies to explain abstract concepts.
- **Be honest**: If something is complex, say so. If there are gotchas, mention them upfront.
- **Interactive**: After the initial explanation, ask what the user wants to explore further.
